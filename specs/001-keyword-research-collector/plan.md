# Implementation Plan: Keyword-Driven Research Collector CLI

**Branch**: `001-keyword-research-collector` | **Date**: 2026-03-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-keyword-research-collector/spec.md`

## Summary

Build a Python 3.10+ CLI tool (`tool.py`) that accepts a seed keyword, queries Google
via SerpAPI using per-domain `site:` queries with `after:2023-01-01` filters, validates
results against an authority domain allowlist and a strict 2023-01-01 date cutoff,
classifies each result by content type (Stats / Survey / Ebook), and writes a clean,
typed Excel (`.xlsx`) and/or CSV output using pandas + openpyxl.

## Technical Context

**Language/Version**: Python 3.10+
**Primary Dependencies**: click, serpapi (google-search-results), pandas ≥ 2.0, openpyxl ≥ 3.1, requests ≥ 2.28, beautifulsoup4, python-dotenv, python-dateutil
**Storage**: Files only — `.xlsx` and/or `.csv` in the working directory
**Testing**: pytest + pytest-mock
**Target Platform**: Cross-platform CLI (Windows / macOS / Linux)
**Project Type**: cli
**Performance Goals**: Complete collection (up to 100 results) within 60 seconds
**Constraints**: `SERPAPI_KEY` env var required; date cutoff ≥ 2023-01-01 non-overridable; no scraping for v1 summaries
**Scale/Scope**: Single-user CLI; up to 100 results per run; ~13 domain queries per run

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Status | Notes |
|---|---|---|
| I. Library-First | ✅ PASS | All concerns covered by established libraries: click (CLI), serpapi (search), pandas/openpyxl (output), requests/urllib3 (HTTP), python-dateutil (dates), python-dotenv (env). `bs4` included for future enrichment but not in critical v1 path. |
| II. Data Relevance | ✅ PASS | Keyword check on title + snippet is a hard gate before any row is written (FR-006). Implemented in `filter.py`, unit-testable in isolation. |
| III. Date Integrity | ✅ PASS | `after:2023-01-01` in SerpAPI queries for pre-filtering + `python-dateutil` parse + `date >= date(2023, 1, 1)` hard check. Unparseable dates excluded. No CLI override below cutoff permitted. |
| IV. Output Quality | ✅ PASS | `pandas.to_excel(engine="openpyxl")` with typed columns; CSV with `utf-8-sig`; atomic write via `os.replace`. |
| V. Resilience | ✅ PASS | `requests.HTTPAdapter` + `urllib3.Retry` for 429/5xx; `CollectionError` wraps exceptions; non-zero exit on failure. |
| VI. Tool Economics | ✅ PASS | SerpAPI gated on `SERPAPI_KEY`; tool degrades gracefully with clear error when key absent; documented in README. |

**Post-design re-check**: All six principles satisfied. No Complexity Tracking violations.

**Structure Decision**: Single project (Option 1). Pure CLI tool with no web front-end
or mobile component. `src/` package contains all domain logic; `tool.py` at root is the
user-facing entry point.
