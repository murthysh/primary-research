# Implementation Plan: Streamlit Web UI

**Branch**: `002-streamlit-web-ui` | **Date**: 2026-03-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/002-streamlit-web-ui/spec.md`

## Summary

Add a Streamlit single-page web app (`streamlit_app.py`) that wraps the existing pipeline (`src/`) with a password gate, keyword form, progress indicator, and in-memory file downloads. Two in-memory serialisation helpers are added to `src/writer.py`. Deploy to Streamlit Community Cloud via `requirements.txt` + `.streamlit/secrets.toml`.

## Technical Context

**Language/Version**: Python 3.10+
**Primary Dependencies**: `streamlit>=1.32.0`, existing `src/` pipeline (serpapi, pandas, openpyxl, click, requests, python-dateutil)
**Storage**: None (all state in `st.session_state`; files served as in-memory bytes)
**Testing**: `pytest` unit tests for new writer helpers; manual smoke test for UI
**Target Platform**: Streamlit Community Cloud (Linux)
**Project Type**: Web app (single-page, server-rendered)
**Performance Goals**: Full collection run completes in < 3 minutes for 12 domains
**Constraints**: No disk writes on server; no raw tracebacks exposed to users
**Scale/Scope**: Small team (< 20 concurrent users)
